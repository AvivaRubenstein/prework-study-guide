CSS

  • CSS RULE TEMPLATE= selector {property: value; property: value;}
  • CSS rules have a selector, which tells you what the rules apply to, followed by brackets containing declarations about the rule,
  • the brackets can contain multiple declarations, each with a property, such as "color" "text-align" "margin" or "padding"
  • followed by a colon, and then the value of that property, for example, "blue" as the value of "color".
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • a * selector means that the rule will apply to all elements on the page
  • you can apply a rule to more than one type of element by writing both selectors followed by a comma before starting the brackets with the rule
  • ex: header, footer {width: 100%; height: 120px;}

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • cd directory-name: change directory
  • git checkout branch-name : move to a different branch
  • to commit changes, first do "git add -A" to commit all files of the working branch
  • then, do "git commit -m "explanation of the changes in this commit" : this commits the changes and puts a message with it
  • git pull origin main: make sure that we are working with the most recent version of the main branch
  • git push origin feature/feature-name : pushes our changes to gitHub
  • Now you can view your new pull request on gitHub!

  • 2. the condition. Example: As long as our var x is less than the length of the array, the loop will keep running: x < shapes.length;

  • format: for (var x = 0; x < shapes.length; x++) { //code block - console.log(shapes[x]); -will print each item in the array to the colsole }
